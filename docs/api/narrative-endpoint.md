# AI Consolidation Narrative endpoint

Phase 2 · Sprint 1 — Narrative. HTTP surface (#150) over the `NarrativeService` (#149).

Read-only against Alvys: the endpoint performs **no** Alvys read or write. The narrative is
generated by the service from already-normalized consolidation-plan data.

## Route

```
GET /api/ai/consolidation/narrative?planId={id}
```

- **Auth:** same Microsoft Entra policy (`AllowedEmailDomain`) as `/api/ltl/*`. An
  unauthenticated caller receives `401` (not `404`).
- **Query:** `planId` — the consolidation plan to narrate.

## Configuration

| Key                  | Type   | Default | Effect                                                              |
| -------------------- | ------ | ------- | ------------------------------------------------------------------- |
| `AI:NarrativeEnabled`| bool   | `false` | Kill switch. When `false` the endpoint returns `404 disabled` and never calls the service. |
| `AI:AzureOpenAI:*`   | object | —       | Azure OpenAI transport settings, consumed by `NarrativeService` (#149), not the endpoint. |

## Responses

| Status | Body                            | When                                                                                   |
| ------ | ------------------------------- | -------------------------------------------------------------------------------------- |
| `200`  | `NarrativeResponse` JSON        | Service returned a non-null narrative.                                                  |
| `404`  | `{ "reason": "disabled" }`      | `AI:NarrativeEnabled=false` (kill switch). Service is not called.                       |
| `404`  | `{ "reason": "plan-not-found" }`| Flag on; service returned `(null, Cached: false)` — the plan id could not be resolved.  |
| `503`  | `{ "reason": "ai-unavailable" }`| Flag on; service returned `(null, Cached: true)` — upstream AI generation failed.       |

### Success headers

| Header         | Value                | Meaning                                        |
| -------------- | -------------------- | ---------------------------------------------- |
| `X-Ai-Source`  | `azure-openai`       | Provenance of the narrative.                   |
| `X-Ai-Cached`  | `true` \| `false`    | Whether the narrative was served from cache (the `Cached` tuple value). |

### `NarrativeResponse` shape

```jsonc
{
  "whyReview":    "string",   // why this plan warrants a human look
  "whatToVerify": "string",   // the specific facts to confirm before acting
  "nextAction":   "string",   // the recommended next step
  "citations":    ["string"]  // plan facts the narrative is grounded in (explainability)
}
```

## Null-response disambiguation (convergence contract with #149)

When the service returns a `null` response and the kill switch is **on**, the endpoint uses the
`Cached` tuple value as the discriminator:

- `(null, Cached: false)` → definitive **plan-not-found** → `404`.
- `(null, Cached: true)`  → transient **AI outage** → `503`.

`NarrativeService` (#149) must honor this convention. See the `TODO(#149)` notes on the stub
interface/DTO in `Features/Ai/Narrative/Contracts/`.
